# CURSOR_TECHMAP

Google Apps Script для автоматизации создания технологических карт в Google Sheets для кабельных сборок и жгутов.

## Что делает решение

- добавляет в Google Sheets меню `Техкарты`
- хранит библиотеку шаблонов в компактном каталоге на скрытых служебных листах
- открывает единую боковую панель с вкладками `Техкарты`, `Материалы` и `Техоперации`
- вставляет выбранный шаблон в текущую активную ячейку рабочего листа
- переносит форматирование, объединения, размеры строк/колонок, тексты, формулы, примечания и изображения вида `IMAGE(...)`
- позволяет технологу после вставки править блок "по месту"
- позволяет технологу самостоятельно нарисовать блок на листе и сохранить его в каталог как новый шаблон без правки кода

В демо-версии создаются 5 шаблонов:

1. Резка провода
2. Резка линейных материалов
3. Зачистка провода (COAX)
4. Монтаж пинов стороны A (COAX)
5. Монтаж коннектора стороны A (COAX)

## Почему новая схема лучше для сотен операций

Да, хранить сотни операций как отдельные листы в одной книге неудобно.

В этой версии шаблоны **не создают отдельные страницы**. Вместо этого:

- в листе `_TC_LIBRARY` лежат метаданные шаблонов
- в листе `_TC_STORE` лежат сами блоки шаблонов в компактном служебном виде
- лист `_TC_CANVAS` используется только как временный холст для генерации демо-шаблонов
- один шаблон = одна запись в каталоге + один блок в скрытом хранилище

Это заметно экономит место в книге и лучше подходит для больших библиотек операций.

## Важное ограничение Google Sheets

Google Apps Script **не позволяет добавлять собственные пункты в стандартное правое контекстное меню** ячейки.

Поэтому в проекте реализован ближайший практический сценарий:

- технолог выбирает ячейку, куда нужно вставить карту или операцию
- открывает `Техкарты -> Открыть библиотеку шаблонов`
- в боковой панели выбирает нужную заготовку
- нажимает `Вставить в текущую ячейку`

Если позже понадобится, этот сценарий можно усилить:

- кнопками прямо на листе
- отдельным листом-навигатором
- HTML-диалогом с предпросмотром
- генерацией карты из нескольких выбранных операций подряд

## Структура проекта

- `DataModel.gs` — формальное описание структуры данных каталога и шаблонов
- `Code.gs` — меню, вставка шаблонов, компактное хранилище, сервисные функции
- `Materials.gs` — загрузка, кэширование и поиск базы материалов из внешней таблицы
- `OperationDatabase.gs` — загрузка, кэширование, поиск и выгрузка базы техопераций
- `Templates.gs` — каталог демо-шаблонов операций
- `Sidebar.html` — прежний интерфейс шаблонов, сохранен как отдельный модуль
- `SaveTemplateDialog.html` — диалог сохранения нового или обновления существующего шаблона
- `MaterialsSidebar.html` — прежний интерфейс материалов, сохранен как отдельный модуль
- `WorkspaceSidebar.html` — единая рабочая панель с вкладками техкарт, материалов и техопераций

## Как развернуть

### Вариант 1. Через editor Apps Script

1. Создайте Google Spreadsheet.
2. Откройте `Extensions -> Apps Script`.
3. Создайте файлы:
   - `DataModel.gs`
   - `Code.gs`
   - `Materials.gs`
   - `Templates.gs`
   - `Sidebar.html`
   - `SaveTemplateDialog.html`
   - `MaterialsSidebar.html`
4. Скопируйте содержимое файлов из этого репозитория.
5. Сохраните проект.
6. Обновите таблицу.
7. При первом запуске выполните `Техкарты -> Проверить доступ к внешним БД`.
8. Если Google запросит разрешения - подтвердите доступ.
9. Затем откройте `Техкарты -> Открыть рабочую панель`.
10. Внутри панели при необходимости обновите базу материалов и базу техопераций соответствующими кнопками.

После этого появятся скрытые служебные листы каталога, но не отдельные листы на каждый шаблон.

## Ограничение по типам файлов в Apps Script

Да, для практического переноса в редактор Google Apps Script вам достаточно файлов:

- `.gs`
- `.html`

Поэтому структура данных вынесена в отдельный файл `DataModel.gs`, а не во внешний JSON/schema-файл.

Файл `appsscript.json` можно не переносить вручную, если не требуется настраивать манифест проекта отдельно.

## Как пользоваться

1. Откройте рабочий лист.
2. Выберите левую верхнюю ячейку, куда нужно вставить блок.
3. Откройте `Техкарты -> Открыть рабочую панель`.
4. На вкладке `Техкарты` найдите подходящую операцию по названию или категории.
5. При необходимости включите галочку `Сокращенный вид`, чтобы увидеть больше карточек.
6. Нажмите `Вставить в текущую ячейку`.
7. Отредактируйте текст, материалы, количество, время и изображения под конкретный проект.

## Работа с базой материалов

Поддерживается внешняя база в Google Sheets со страницами:

- `COMPCON`
- `COMPCOAX`
- `COMPTERM`
- `COMPWIRE`
- `COMPACCESS`

Инструмент загружает данные из этих листов по колонке `Поисковый тег`, а затем:

- складывает нормализованный индекс в скрытый лист `_MAT_DB`
- сохраняет дату последней загрузки в `_MAT_META`
- держит быстрый кэш в памяти Apps Script

Это нужно, чтобы технолог работал уже в текущем документе техкарт, а не прыгал между книгами.

### Как обновить базу материалов

1. Откройте документ с техкартами.
2. Если разрешения еще не выдавались, выполните `Техкарты -> Проверить доступ к внешним БД`.
3. Затем выберите `Техкарты -> Обновить базу материалов`.
4. Откройте `Техкарты -> Открыть рабочую панель`.
5. Перейдите на вкладку `Материалы`.
6. При необходимости нажмите `Обновить базу из источника`.
7. Дождитесь сообщения об успешной загрузке.

### Что умеет вкладка материалов

- поиск по всем загруженным поисковым тегам
- автодополнение по типу, производителю, поставщику
- генератор по шаблону артикула с переменной `XX`
- генератор коаксиальных сборок
- корзина материалов
- выгрузка корзины в активный лист
- размножение значений в столбец по количествам
- сокращенный вид для более плотного списка результатов и карточек

## Как самостоятельно создавать новые шаблоны

1. На обычном рабочем листе вручную отрисуйте эталонный блок так, как он должен выглядеть.
2. Выделите весь диапазон шаблона.
3. Откройте `Техкарты -> Сохранить выделение как шаблон`.
4. Укажите:
   - название,
   - категорию,
   - описание.
5. Сохраните шаблон.

После этого шаблон появится в боковой панели и станет доступен для вставки в другие проекты.

## Что именно сохраняется в шаблон

Сохраняются:

- значения

- фон

Основные сущности:

  - `width`

  - `sourceRange`

## Как обновлять существующий шаблон

Теперь доступны оба сценария:

1. создать новый шаблон из выделенного диапазона;
2. выбрать существующий шаблон в диалоге и обновить его текущим выделением.

Практически это означает, что можно:

1. вставить существующий шаблон на рабочий лист;
2. исправить его до нужного вида;
3. снова открыть `Техкарты -> Сохранить выделение как шаблон`;
4. выбрать существующий шаблон в списке и перезаписать его.

## Как пополнять библиотеку

Теперь библиотеку можно пополнять без правки кода:

- демо-шаблоны ставятся автоматически
- пользовательские шаблоны добавляются из выделенного диапазона
- каталог можно фильтровать по категориям

Дальнейшее развитие, которое логично сделать следующим этапом:

- предпросмотр шаблона перед вставкой
- отдельный лист-реестр шаблонов с кнопками управления
- категории: COAX, сборка, разделка, пайка, маркировка, упаковка
- шаблоны по типам изделий: BNC, SMA, жгут, силовой кабель, LAN
- хранение ссылок на корпоративные изображения и инструкции

## Архитектурная идея

Решение специально сделано без внешней базы:

- каталог = служебный лист `_TC_LIBRARY`
- данные шаблонов = служебный лист `_TC_STORE`
- временный холст генерации = служебный лист `_TC_CANVAS`
- кэш материалов = служебные листы `_MAT_DB` и `_MAT_META`
- пользовательский интерфейс = sidebar

Это упрощает внедрение на производстве:

- не нужна отдельная инфраструктура
- технолог работает прямо в знакомом Google Sheets
- шаблоны можно переносить копированием книги
- сотни операций не раздувают книгу сотнями вкладок

## Что можно сделать следующим шагом

Если захотите, следующим сообщением я могу доработать систему до одного из вариантов:

1. **Конструктор техкарты из операций**  
   Пользователь выбирает несколько операций, а скрипт собирает готовую карту по порядку на новом листе.

2. **Редактирование и версионирование шаблонов**  
   Технолог может не только создавать новый шаблон, но и обновлять существующий по ID или сохранять новую версию.

3. **Справочник материалов и норм времени**  
   Скрипт подставляет стандартные материалы, длины, нормы времени и названия операций автоматически.

4. **Настоящий production-вариант под ваш процесс**  
   Адаптация структуры именно под ваши технологические карты, форматы блоков, изображения и правила именования.
