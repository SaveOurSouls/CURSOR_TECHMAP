# TECHMAP — Google Apps Script аддон для техкарт кабельных жгутов

Аддон для Google Sheets для производственных инженеров ООО «ЗЭП ФРАКСИС».
Управляет **технологическими картами** сборки кабельных жгутов: библиотека шаблонов,
база техопераций и **генератор техкарт сборки**, который собирает цепочку карт
(резка → опрессовка → монтаж → свивка) прямо из спецификации изделия.

---

## Возможности

- **Библиотека шаблонов** — сохранение выделенного диапазона как шаблона (значения,
  формулы, форматирование, объединения, размеры, картинки) и вставка в новый лист.
- **База техопераций** — синхронизация из внешней таблицы-источника, кеш, браузер в сайдбаре.
- **Генератор техкарт сборки** — по спецификации изделия (BOM) генерирует цепочку
  заполненных техкарт; авто-определяет стороны, кол-во проводов, пары свивки и времена.

## Генератор техкарт сборки

`generateAssemblyTechCards(config)` — по спецификации (BOM) изделия создаёт лист на каждую
операцию с шаблоном, заполняет его структурно и протягивает результат по цепочке.

### Цепочка операций

`Резка → Опрессовка A → [Свивка] → Монтаж A → Опрессовка B → [Свивка] → Монтаж B`

Типы операций: `cutWire`, `prsTermA`/`prsTermB` (опрессовка), `insTermA`/`insTermB` (монтаж),
`twist` (свивка). Добавление нового типа — скилл `/new-optype`.

### Структурный фил (без плейсхолдеров)

Ячейки находятся по **текстовым паттернам**, а не по `{{ТОКЕНАМ}}`: `Комплектующие`,
`Полуфабрикат`, `Результат`, `Расчетное время`, колонки `Допуск` / `№ Прог.`, строки `Контроль`.
Результат протягивается на «вход» следующего листа; на последнем листе полуфабрикат
переименовывается в изделие (`assemblyName`).

### Авто-определения из BOM

- **Стороны A/B** — по заложенным разъёмам: один разъём ×2 → зеркало; два разных → A/B.
- **Кол-во проводов** — по соотношению норм: `round(норма / минимальная норма)`
  (кратная норма = несколько одинаковых проводов).
- **Свивка (TWIST)** — провод с дробным излишком сверх целого кратного базовой нормы
  (= запас на свивку) попадает в пару; кратные нормы (кол-во) свивку не триггерят.
- **Пары свивки** — режим «Парами» (конструктор) или «Все вместе».

### Запас на свивку при резке

Свиваемые провода режутся длиннее по геликоидальной геометрии:
`L_рез = L · √(1 + (π·d / p)²)`, где `p` — шаг свивки (один полный оборот, как S1 на чертеже),
`d` — Ø провода с изоляцией. Ø подставляется автоматически из листа `СПР.КОАКС`
по AWG (или ГОСТ-сечению) + марке провода; ручное поле — fallback/override.

### Подсветка достоверности полей

В операциях: 🟢 зелёный — данные из БД (совпадение с БД.ОП), 🟡 жёлтый — вручную/без
совпадения (проверить), 🔴 красный — пусто/не подтянулось.

### Прочее

- Кнопка **«Удалить созданные»** — сносит листы последней генерации (быстрые итерации).
- Времена операций берутся из БД.ОП по коду шаблона; время свивки = `время · число свивок · партия`.

## Картинки

- При сохранении шаблона картинки кешируются в Drive (папка `_TC_IMAGE_CACHE`) —
  ускоряет вставку.
- Получение blob: `getBlob()` → `getUrl()`+`UrlFetchApp` → XLSX-unzip (fallback для GAS 2024+).
- Файлы кеша расшариваются «по ссылке — просмотр», чтобы картинки подтягивались
  у всех пользователей таблицы (а не только у владельца Drive-файлов).

## Кастомные формулы (в шапках техкарт)

```
=SHEETNAME()   → полное имя листа
=SHEET_CODE()  → часть до " | "    ("CUT_WIRE_auto | Резка" → "CUT_WIRE_auto")
=SHEET_TYPE()  → часть после " | "  ("CUT_WIRE_auto | Резка" → "Резка")
```
Имена листов/шаблонов: `КОД | Тип` (например `CUT_WIRE_auto | Резка`). Код = `opNumber` в БД.ОП.

## Развёртывание

Через [clasp](https://github.com/google/clasp):

```bash
clasp push --force   # полная синхронизация (push = 12 файлов)
```

`clasp push --force` удаляет из GAS файлы, отсутствующие локально. Деплоить после каждой
правки `.gs`/`.html`. После бампа `schemaVersion` — выполнить «Синхронизировать базу
техопераций» из меню (или открыть генератор — синхронизирует сам).

## Соглашения

- Приватные функции оканчиваются на `_` (`fillSfIn_`). Комментарии — по-русски, идентификаторы — по-английски.
- Квота GAS: батч-чтение/запись (никаких `getValue`/`setValue` в циклах).
- Чистая логика изолирована от Google-сервисов и покрыта локальными тестами.
- Технические десятичные — через запятую (`formatDecimalComma_`); снапшот читается `getDisplayValues`.
