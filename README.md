# TECHMAP — Google Apps Script аддон для работы с техкартами

Аддон для Google Sheets, предназначенный для производственных инженеров.
Позволяет создавать, хранить и переиспользовать **техкарты** (шаблоны технологических карт),
а также работать с базами техопераций и материалов прямо из таблицы.

---

## Архитектура

```
appsscript.json         — манифест (scopes, V8, Sheets API v4)
Code.gs                 — точка входа: меню, кастомные формулы, публичное API
TemplateStore.gs        — внутренняя реализация: каталог, хранилище, копирование, картинки
OperationDatabase.gs    — БД техопераций (синхронизация из внешней таблицы + кеш)
Materials.gs            — БД материалов (синхронизация из внешней таблицы + кеш)
WorkspaceSidebar.html   — боковая панель (три раздела: шаблоны, техоперации, материалы)
SaveTemplateDialog.html — модальный диалог сохранения шаблона
```

---

## Системные листы (скрытые)

Аддон создаёт служебные листы автоматически и скрывает их от пользователя.

| Лист | Назначение |
|---|---|
| `_TC_LIBRARY` | Каталог шаблонов (id, title, storeRow, размеры, JSON картинок) |
| `_TC_STORE` | Физическое хранилище содержимого шаблонов (значения + форматирование) |
| `_TC_TECHOPS_META` | Мета-информация БД техопераций (источник, дата, версия схемы) |
| `_TC_TECHOPS_DB` | Данные БД техопераций (13 колонок, локальная копия) |
| `_TC_MATERIAL_META` | Мета-информация БД материалов |
| `_TC_MATERIAL_DB` | Данные БД материалов (10 колонок, локальная копия) |

## Кастомные формулы для ячеек

Используются в шапках техкарт, привязывая текст к имени листа.

```
=SHEETNAME()       → полное имя листа
=GET_NAME(любое)   → полное имя листа (совместимость с =GET_NAME(Таблица1[Индекс]))
=SHEET_CODE()      → часть до " | "    ("CUT_WIRE_auto | Резка" → "CUT_WIRE_auto")
=SHEET_TYPE()      → часть после " | " ("CUT_WIRE_auto | Резка" → "Резка")
```

Соглашение об именовании листов: `КОД_ОПЕРАЦИИ | Тип` (например, `CUT_WIRE_auto | Резка`).

## Модуль шаблонов (Code.gs + TemplateStore.gs)

### Как работает хранилище

- `_TC_LIBRARY` — каталог с метаданными каждого шаблона (id, title, storeRow, storeColumn, height, width, rowHeightsJson, columnWidthsJson, imagesJson).
- `_TC_STORE` — один большой лист, где диапазоны шаблонов лежат блоками друг под другом начиная со строки `storeRow`.
- При сохранении блок дописывается после последнего; при перезаписи занимает тот же слот если размер не изменился.
- При вставке (`insertTemplate`) создаётся новый лист с именем шаблона, содержимое копируется начиная с B2.

### Копирование диапазонов (`copyRangePreservingFormulas_`)

Многоуровневый fallback чтобы сохранить формулы, форматирование, объединения, картинки:

1. **Sheets API v4 `copyPaste`** — основной путь, сохраняет in-cell изображения
2. **`range.copyTo(PASTE_NORMAL)`** — если Sheets API недоступен
3. **`setValues` + `PASTE_FORMAT`** — если copyTo упал
4. После любого пути — `applySourceFormulasCellwise_` накладывает формулы поячеечно
5. `copyInCellImageValues_` через Sheets API явно переносит `userEnteredValue.imageValue`

### Обработка картинок (over-grid)

Цепочка получения blob'а для over-grid изображения:
1. `img.getBlob()` — старый GAS API
2. `img.getUrl()` + `UrlFetchApp` с OAuth токеном — GAS 2024+
3. Скачивание XLSX → unzip → парсинг drawing XML (`buildXlsxImageMap_`) — последний резерв

При сохранении шаблона картинки также кешируются в Google Drive (папка `_TC_IMAGE_CACHE`), что ускоряет последующую вставку.

### Валидация

- Запрещены диапазоны с частично захваченными объединёнными ячейками.
- Нельзя сохранять с системных листов (`_TC_*`, `_TPL_*`).
- Если выделение изменилось с момента открытия диалога — ошибка с предложением переоткрыть.

## Модуль техопераций (OperationDatabase.gs)

### Источник данных

Внешняя таблица: `1W3VK9Fw71lYdw1Klcsn_za5-2EhvLoXIAKZVYOCnKcs`

Четыре вкладки источника:

| Ключ | Лист | Описание |
|---|---|---|
| `ob` | БД.ОБ | Плоский список (поле "Для базы") |
| `op` | БД.ОП | Операции: номер, название, времена, расходы |
| `ter` | БД.ТЕР | Терминалы/разъёмы: комплектующая, аналог, серия, производитель |
| `coax` | БД.КОАКС | Коаксиальные изделия: артикул, программа, провод, D/L размеры |

### Кеширование

- `CacheService.getDocumentCache()` чанками по 80 000 символов, TTL 6 часов.
- Версия схемы `schemaVersion: 3` — при несовпадении автосинхронизация.
- Локальная копия в `_TC_TECHOPS_DB` / `_TC_TECHOPS_META` — резерв при холодном кеше.

### Заголовки — автоопределение

`detectTechOperationsHeaderRow_` сканирует первые 12 строк листа и выбирает строку с наибольшим числом совпадений с ожидаемыми псевдонимами колонок (например, для `op` ищет "номер"/"number", "название"/"name", "время операции" и т.д.).

## Модуль материалов (Materials.gs)

### Источник данных

Внешняя таблица: `1NExDzeG-vw3zY_ooeXoxRffIARh2wbLJNAHT3FU_Ig8`

Листы-источники: `COMPCON`, `COMPCOAX`, `COMPTERM`, `COMPWIRE`, `COMPACCESS`

Ожидаемые колонки (с псевдонимами): поисковый тег, артикул, тип, производитель, поставщик.

### Дедупликация

Записи дедуплицируются по `normalizedTag` (lowercase, без пробелов). Если поисковый тег пуст — собирается из `артикул | тип | производитель | поставщик`.

## Боковая панель (WorkspaceSidebar.html)

Три главных вкладки: **Техкарты** / **Техоперации** / **Материалы**.

Активная вкладка сохраняется в `localStorage` и восстанавливается при следующем открытии.

### Вкладка Техкарты
- Поиск по названию/категории/описанию, фильтр по категории
- Карточки шаблонов: Вставить / Удалить
- Автоопрос версии каталога каждую секунду — список обновляется без перезагрузки

### Вкладка Техоперации

| Подвкладка | Структура | Что вставляется |
|---|---|---|
| БД.ОБ | Плоский список | Значение поля "Для базы" |
| БД.ОП | 2 уровня: Название → Номер | Номер \| Название, времена, расходы |
| БД.ТЕР | 3 уровня: Производитель → Серия → Комплектующая | Комплектующая, аналог, серия, производитель |
| БД.КОАКС | 4 уровня: Провод → Тип → Производитель → Артикул | Артикул, программа, D1–D3, L1–L3 |

Выбранные записи попадают в **очередь** → выгружаются в таблицу одной операцией начиная с указанной ячейки.

### Вкладка Материалы

| Подвкладка | Назначение |
|---|---|
| Поиск | Полнотекстовый поиск по базе → добавление в корзину |
| Шаг XX | Генератор артикулов по шаблону с переменной XX (напр. `PHR-XX`) |
| Коаксиал | Подбор пар разъёмов по типу и производителю с постоянными позициями |
| Копии | Реплицирование: value × count строк → выгрузка столбцом |

**Корзина** — буфер для накопления позиций перед выгрузкой в таблицу.

## Развёртывание

1. Открыть Google Sheets → Расширения → Apps Script.
2. Загрузить все файлы проекта.
3. Services → добавить Google Sheets API (Sheets v4).
4. Обновить страницу таблицы — появится меню **Техкарты**.
5. Техкарты → Открыть рабочую панель.
6. При первом открытии каждой БД нажать **Обновить базу из источника**.

## Известные нюансы

- **Скрытые листы и API**: операции `clear`, `copyTo` на скрытом листе иногда дают "Ошибка службы: Таблицы". Функция `runWithSheetVisible_` временно показывает лист на время операции.
- **setValues на большом диапазоне**: при падении режется на полосы по 25 строк → построчно → поячеечно.
- **Картинки GAS 2024+**: `getBlob()` перестал работать для новых типов изображений, fallback через `getUrl()` + UrlFetchApp, последний резерв — XLSX download.
- **Версионирование каталога**: `bumpCatalogVersion_` пишет timestamp в UserProperties; сайдбар опрашивает версию каждую секунду и обновляет список без перезагрузки.
- **Компактификация STORE**: `compactifyStore_` пакует блоки без пустых промежутков. Вызывается не автоматически — только явно, чтобы операция сохранения оставалась быстрой.
